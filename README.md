# GeocodeEXE

Теперь приложение собрано в один переносимый файл: `geocode_exe.py`.

## Как передавать

- Для разработчика/пользователя с Python: передайте только `geocode_exe.py`.
- Для пользователя без Python: соберите один `GeocodeEXE.exe` и передайте только этот `.exe`.

## Запуск одного Python-файла

```bash
python geocode_exe.py
```

CSV/TXT и запросы к сервису геокодирования работают без сторонних Python-библиотек. Для открытия/сохранения Excel `.xlsx/.xlsm` нужен `openpyxl`:

```bash
python -m pip install openpyxl
```

Для сборки EXE используйте отдельный минимальный список зависимостей из `requirements-exe.txt`: он содержит `pyinstaller` для сборки, `openpyxl` для Excel и `s2sphere` для генерации S2-тайлов по полигонам. Зависимость `et_xmlfile` будет установлена автоматически как обязательная зависимость `openpyxl`.

## Сборка одного EXE

```bash
python -m pip install --upgrade -r requirements-exe.txt
pyinstaller --onefile --windowed --name GeocodeEXE --hidden-import=openpyxl geocode_exe.py
```

Готовый файл будет здесь:

```text
dist/GeocodeEXE.exe
```

После сборки можно передавать пользователю только `dist/GeocodeEXE.exe`; папки проекта и Python ему не нужны.

Если собираете другой входной файл, например `kml_generator_app.py`, оставьте тот же принцип: сначала установите `requirements-exe.txt`, затем добавьте `--hidden-import=openpyxl` в команду PyInstaller:

```bash
python -m pip install --upgrade -r requirements-exe.txt
pyinstaller --onefile --windowed --hidden-import=openpyxl kml_generator_app.py
```

Опция `--hidden-import=openpyxl` нужна, потому что приложение подключает Excel-поддержку динамически, а PyInstaller не всегда видит такие импорты автоматически.

## Что умеет интерфейс

- Загружать `.xlsx`, `.xlsm`, `.csv`, `.txt`.
- Определять координаты по адресу.
- Определять адрес по координатам.

- Работать через вкладки: отдельно геокодирование и отдельно генерация S2-тайлов по полигонам.
- Загружать свой полигон из GeoJSON/JSON, KML или CSV/TXT с координатами/WKT и формировать S2-тайлы через `s2sphere`. В результат попадают только тайлы, чей центроид строго внутри полигона; исходные колонки полигона сохраняются, а к ним добавляются `S2 cell id`, широта и долгота центроида тайла.
- Выбирать уровень S2-тайлов в выпадающем списке с примерным размером, например `13 (~1 000 × 1 000 м)`.
- Работать через вкладки: отдельно геокодирование и отдельно проверка центроидов по полигонам.
- Загружать полигон из GeoJSON/JSON, KML или CSV/TXT с координатами и отмечать, какие центроиды из таблицы попали внутрь полигона.
- Показывать предпросмотр первых 200 строк.
- Сохранять результат в Excel, CSV или TXT.
- Использовать встроенные настройки сервиса без показа технических URL пользователю.
- Добавлять в результат понятные колонки: `Найденный адрес`, `Широта результата`, `Долгота результата`.

## Ошибка HTTP 400 при геокодировании

Если при направлении «адрес → координата» сервис возвращает `HTTP 400: Bad Request`, обычно это означает, что внешний сервис не принял конкретный запрос: адрес слишком длинный, содержит переносы/лишние пробелы или сервис отклонил формат тела запроса.

В приложении запросы к сервису нормализуются перед отправкой: лишние пробелы и переносы удаляются, строка адреса ограничивается 300 символами, а результат по каждой строке получает колонку `Ошибка геокодирования`. Поэтому один проблемный адрес больше не останавливает обработку всего файла — причину можно увидеть в результирующей таблице.
